HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML elements work together to build the frame of a webpage
  • As a developer you should think of HTML First!
  • HTML elements are often composed of opening and closing tags represented by angled brackets
  • Each html element performs a unique role however two elements stand above all to understanding how the structure works.
  • One of the three cornerstone languages in web develpoment, they make up the structure of a webpage.
  • The "body" element contains the content that is hsown to the user. All the elements that you want the user to see and interact with need to be placed within the "body".
  • The "head" element contains info, or metadata, that helps the browser render the page correctly. Elements included within the "head" are not visible when a user looks at a webpage.
  • THE FOLLOWING ARE BODY ELEMENTS:
  • "h1" represents the header and there can be multiple on a webpage.
  • "p" element is a block of text or paragraph.
  • "ul", "ol", and "li" elements represent unordered lists, ordered lists, and list items.
  • "img" element contains info about images displayed on the webpage. This can contain various attributes but the src attribute is the most important because it shows the location of the image.
  • "br" represents a break in text.
  • "a" element represents an anchor and it creates a link to a webpage.
  • THE FOLLOWING ARE HEAD ELEMENTS:
  • The "base" element defines the base URL for a webpage.
  • The "link' element connects an external resource to the HTML document.
  • The "meta" element defines metadata such as the character set, description, keywords, author, and viewport.
  • The "script" element embeds client-side scripts in the HTML document.
  • The "charset" element sets our character encoding.
  • The "http-equiv" element sets an HTTP header for content.
  • The "viewport" element sets the default view for our webpage.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are three ways to stle a webpage using css; inline css, internal css style sheet, and external css style sheet.
  • The most common way to style a webpage is by using an external css style sheet.
  • External style sheets are so popular among developers because it allows them to keep their css elements seperate.
  • Developers use inline css if they want to style an element directly in the HTML File.
  • Inline is only used for quick and permanent changes since they are less flexible.
  • Internal CSS style sheets contain css rules in the head section of the HTML File.
  • Internal CSS style sheets rules only apply to that specific webpage where you can configure css classes and ids.
  • CSS RULE div {color: blue; text-align: justify;}
  • The margin property acts much like a margin in a word processing document
  • A margin property indicates how much space we want around the outside of an element.
  • The padding property adds space around the content inside an element.
  • Each box consists of margins outside the element, and the element's borders, padding, and content.
  • .CARD ELEMENT
  • The period preceding card designates this as a class selector. Any HTML element assigned to the class, card, will gain these property assignments from the CSS.
  • The first declaration addresses the width of the element at 80% of the available width.
  • The margin property assigns 40px to the top and bottom, and the auto value to the left and right.
  • The border property assigns a border with a width of 5px that is solid and gray.

Git

  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main: checks to see if your local branch is up to date with base branch in GitHub
  • git add -A: stages new code to be pushed to GitHub
  • git commit -m "your message here": creates a snapshot of current code and will update once pushed
  • git push origin feature/branch-name: pushes code to GitHub.

  • "cd" navigates to a directory.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript in a gyst is a programming language that allows users to interact with a website
  • JavaScript is a dynamic language with dynamic types. Variables in JavaScript are not directly associated with any particular value type, and any variable can be assigned (and re-assigned) values of all types
  • The .js file extension indicates that this file is a JavaScript file.
  • To declare a variable in JavaScript we use var, after that keyword we give the variable a name, it must be unique because it is how you reference the information stored.
  • We use an assignment operator = to assign the specific data to be stored. var shape = circle.
  • Each line in java script ends in a semicolon.
  • console.log method is used to ouput a message to the web console by adding an argument.
  • The word console refers to a test environment that developers use to check out their code. The second part, .log(), is a method, which means a set of instructions that can be executed by a computer. The .log() method will output whatever we add inside the parentheses to the console.
  • The script.js file is linked in the body element.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript runs from the first statement at the top of the page, down to the last statement at the bottom of the page.
  • We define topics AFTER we tell the computer to log it in the console.
  • An "if" statement is a way of introducing decision-making into our code.
  • If the computer encoutners a code that is "truthy" it will execute it, if it isnt "truthy" then it will not.
  • If you are adding more conditions you would us an "else if".
  • The strict equality operator "===" checks to see if two values are equal, and then returns a boolean result to true if the values are equal and false if they are not.
  • An Array is a single variable that is used to hold a group of data.
  • Strings need to be contained within quotation marks.
  • Arrays are useful tools for storing multiple items with different data types in a single variable, but not all arrays must have multiple data types. The array that we will add to our project only contains strings.
  • The location of the data item in the array is identified with a unique number called an index.
  • A "for" loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • A "for" loop contains three important statements; starting point for our loop(var x = 0;), Condition(xshapes.length;), the final statement allows the array to iterate over each item(x++).
  • A function is simply a set of instructions that tell the computer how to do a task.
  • to create a function we use the "function" keyword.
  • To declare a function, we use the function keyword and then give our function a unique name followed by parentheses. Then we add curly brackets {} that contain the code for the task we want to perform. In our example, we created two separate functions: one for our for loop and one for our if statement.
  • A function's name is followed by parentheses (). Later on, we will define parameters that can be placed inside the parentheses. For now, just remember that it is important to include the parentheses whenever you write a function.
